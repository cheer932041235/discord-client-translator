---

## 1. 背景

用户开始使用 Discord 观察 AI 相关高质量社群，例如 Windsurf、AI 编程工具、LLM/RAG、Agent、生图和本地模型社区。Discord 高质量信息密度较高，但大量频道为英文实时讨论，直接阅读会增加认知负担。

用户希望只使用 Discord 桌面客户端，而不是网页版。因此本项目聚焦客户端侧阅读增强：在 Discord 客户端消息下方显示中文译文。

本项目不是爬虫、不是群控、不是 selfbot，也不是账号操作工具。它只做本地 UI 增强和翻译显示。

---

## 2. 一句话目标

在 Discord 桌面客户端中，为当前可见英文消息自动生成中文译文，并显示在原消息下方，翻译后端使用 DeepSeek 官方 API，默认模型为 `deepseek-v4-flash`。

---

## 3. 目标用户

第一阶段只服务用户本人：

- 使用临时 Discord 观察号浏览 AI 社群。
- 主要阅读英文频道中的技术讨论、公告、showcase、support 信息。
- 希望快速判断消息价值，而不是逐句手动复制翻译。

暂不以公开发布为目标，但目录和配置方式应按可维护工具项目设计。

---

## 4. 使用边界与安全原则

### 4.1 明确要做

- 读取当前客户端界面中已经渲染的可见消息文本。
- 判断消息是否需要翻译。
- 调用本地翻译服务生成中文译文。
- 把译文作为 UI 附加内容插入到原消息下方。
- 对已翻译消息做本地缓存，避免重复请求。
- 支持手动触发、自动触发、频道级开关。

### 4.2 明确不做

- 不读取、不保存、不导出 Discord token。
- 不读取密码、2FA、cookie 或登录凭据。
- 不发送消息、不自动回复、不加群、不加好友。
- 不批量抓取服务器历史消息。
- 不绕过 Discord 权限或访问当前用户不可见内容。
- 不做广告、群发、自动互动等违反平台规则的行为。

### 4.3 风险定位

本工具属于客户端 UI 增强，有 Discord ToS 灰区风险。第一阶段只用于用户个人观察号，不建议绑定主身份使用。

## 5. 功能需求

### 5.1 MVP 功能

MVP 只做最小可用版本：

1. 在 Discord 客户端中识别当前频道可见消息。
2. 提取消息文本，过滤纯中文、纯表情、纯链接、过短文本。
3. 对英文或中英混合技术消息发起翻译请求。
4. 在消息下方插入中文译文。
5. 同一条消息不重复翻译。
6. 默认启用本地缓存，避免滚动和重复渲染导致重复调用 API。
7. 翻译失败时显示简短错误提示，不影响 Discord 原界面。
8. API Key 不出现在客户端插件代码里。

### 5.2 第一版增强功能

- 自动翻译当前可见消息和新增可见消息。
- 频道级开关：当前频道启用/暂停自动翻译。
- 全局开关：启用/暂停插件翻译功能。
- 翻译结果本地缓存。
- 保留技术术语：Windsurf、Cascade、MCP、RAG、LLM、SWE、repo、PR、issue、refactor、terminal 等。
- 代码块、命令行、URL、用户名不翻译。
- 长消息可以输出“中文翻译 + 一句话摘要”。

### 5.3 后续可选功能

- 右键消息菜单：翻译、总结、解释术语。
- 频道摘要：对当前屏幕或最近 N 条可见消息生成摘要。
- 收藏高价值消息到 Obsidian（暂不进入 MVP，仅列入后续待办）。
- 与 Browser Intelligence 联动，沉淀 AI Discord 社群情报。
- 支持多翻译后端：DeepSeek-V4-Flash、传统翻译 API、本地 Ollama。

## 6. 非功能需求

### 6.1 性能

- 翻译请求应异步执行，不阻塞 Discord UI。
- 同时翻译请求需要限流，避免 API 爆量。
- 滚动频道时不应卡顿。
- 默认只处理可见区域和新出现的消息。

### 6.2 稳定性

- Discord DOM/React 结构变化时，插件不能导致客户端崩溃。
- 插入译文前检查是否已存在译文节点。
- 所有外部调用都要有 timeout。
- 翻译服务不可用时，插件降级为不显示译文。

### 6.3 隐私

- 默认不保存原文和译文到长期文件。
- 如启用缓存，缓存只保存在本地，并可一键清理。
- 不上传非必要上下文。
- 不把服务器 ID、频道 ID、用户名作为必须上传内容。

### 6.4 可配置

配置应包括：

- 本地翻译服务地址。
- 模型名，默认 `deepseek-v4-flash`。
- 是否自动翻译。
- 最小消息长度。
- 并发数限制。
- 缓存开关。
- 术语保留表。

## 7. 技术路线候选

### 路线 A：Vencord 插件 + 本地翻译服务

```text
Vencord 插件
  → 监听 Discord 消息组件/DOM
  → 调用 http://127.0.0.1:xxxx/translate
  → 插入译文 UI

本地 Node.js 服务
  → 读取 config.json 或环境变量
  → 调用 DeepSeek 官方 API
  → 返回译文
```

优点：

- 更贴近 Discord 客户端插件生态。
- UI 集成体验好。
- 适合做按钮、频道开关、缓存状态。

缺点：

- 有客户端 Mod 灰区风险。
- 需要调研 Vencord 插件开发流程。
- Discord 更新可能导致适配成本。

### 路线 B：BetterDiscord 插件 + 本地翻译服务

优点：

- 插件开发资料较多。
- DOM 注入和样式插入直观。

缺点：

- 安全口碑和生态质量参差。
- 插件权限较大。
- 后续维护不一定优于 Vencord。

### 路线 C：CDP/外部自动化注入

使用已有 Browser Engine 或 Electron/Chrome DevTools Protocol 对 Discord 客户端页面注入脚本。

优点：

- 与现有 `browser-engine` 能力相关。
- 便于调试 DOM 和消息结构。

缺点：

- 桌面 Discord 是否暴露可连接调试端口不稳定。
- 启动方式复杂。
- 不适合作为长期日常工具主路线。

### 已确认推荐

第一阶段推荐：**路线 A：Vencord 插件 + 本地翻译服务**。

原因：它最符合“只用客户端”的需求，同时可以把 API Key 隔离在本地服务中，客户端插件只访问 localhost。

## 8. API 翻译后端设计

### 8.1 推荐调用链

```text
Discord 插件
    ↓ localhost HTTP
本地 translation-service
    ↓ 可配置上游
DeepSeek 官方 API，默认模型 deepseek-v4-flash
```

### 8.2 API Key 管理

- 不在插件里写 Key。
- 不在 Git 提交真实 `config.json`。
- 提供 `config.example.json`。
- 优先从环境变量 `DEEPSEEK_API_KEY` 读取。
- 本项目主路径不复用 SophNet/api-gateway；后续可作为备用后端。

### 8.3 翻译 Prompt 草案

```text
你是面向中文开发者的 Discord AI 技术社区翻译器。
请把下面英文消息翻译成自然中文。
要求：
1. 保留专有名词，如 Windsurf、Cascade、MCP、SWE、RAG、LLM、agent、repo、PR、issue。
2. 保留代码、命令、URL、用户名、频道名。
3. 语气贴近日常技术社区，不要过度书面。
4. 如果原文是中文或不需要翻译，返回空字符串。
5. 只输出中文译文，不要解释。

消息：
{{text}}
```

### 8.4 长消息 Prompt 草案

```text
请翻译下面 Discord 技术社区长消息。
输出格式：
译文：...
要点：用一句话说明它在说什么。

要求：保留术语、代码、命令、URL、用户名。

消息：
{{text}}
```

## 9. 消息识别与去重策略

### 9.1 消息识别

候选来源：

- DOM 中的消息文本节点。
- Discord React 组件结构。
- Vencord 可用的消息/组件 patch API。

优先级：

1. 插件生态提供的消息对象。
2. 稳定 DOM selector。
3. MutationObserver 兜底。

### 9.2 去重 Key

建议使用：

```text
channelId + messageId + contentHash
```

如果 MVP 暂时拿不到 messageId，则用：

```text
messageElement.dataset + textHash
```

### 9.3 跳过规则

跳过以下消息：

- 空文本。
- 纯中文。
- 纯 emoji。
- 纯 URL。
- 少于 8 个英文字符。
- 代码块占比过高。
- 已经插入译文的消息。

## 10. UI 需求

### 10.1 译文显示

译文应显示在原消息下方，样式低调，不喧宾夺主。

建议样式：

```text
中文译文使用较小字号、灰色边框或左侧竖线、浅色背景。
```

### 10.2 状态显示

每条消息可显示：

- 翻译中...
- 翻译失败，点击重试
- 已翻译

MVP 可只显示译文和失败提示。

### 10.3 控制入口

第一版可以使用插件设置页：

- 开启/关闭自动翻译。
- 当前频道启用/停用。
- 清空缓存。
- 测试翻译服务。
